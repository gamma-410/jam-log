Go言語でDDSKに参加した話

今宵のお天気はどうでしょう。こんばんは✨ @gamma410です。
最近はGoを毎日書くことにハマっています...

本題

さて、今回。@Comamoca さんのGitHubリポジトリ「Awesome-DDSK」にPRして参加してきました...
私は血迷っていたのか、ノリで「GoでDDSKするか()」という呟きをしてしまい、無知で知識皆無なのにも関わらずコードを書いていくことになりました。
無事にPRできた時マジでホッとしました。

難しかったこと・ハマったこと

Go初心者の私がハマったところを紹介します。
- ddsk[2] string = [2]string {"ドド", "スコ"}
まず、ここでハマりました。
ここでの学びは、配列はconstできないと言うことです。Goの配列・スライスは特徴的ですよね。そのため、
array := [int]<型> {<要素>, <要素>, <要素>, ...}
こんな感じです。intとあります。配列内の個数は固定です。なので、私のような初学者は「これってconstできるんじゃね？」ってなるんです。しかし、array 自体は可変するものなのです。マジでハマりました。stackoverflowは神です、おすすめ。(海外の方の情報がめっちゃ使えるよ)

  • len()
    これもマジでハマりました。
    なんなんですか、マジで。
    いつもPythonと仲良くしている私。Pythonくんの len() は文字数、長さを返してくれます。
    しかーし、Goくんの len() は byte長 で返してきます。
    つまり、アルファベット・数字なら良いんです。1byteなので。しかし、我々の使用している日本語、ジャパニーーーーズは、3byteなんです。(1文字)。
    いやーなんなんですかね。 よくわからなくて、無限ドドスコに陥っていましたよ、まったくぅ、もうっ！！()
    3byte 必要ということを todo というコマンドを書いた時(まだ未完成)に学んだので役に立ちました。
    勘でゴニョゴニョしてたらできたので、ここはもう少し調べて理解していきたいです。
    解決法として思いついたのは、"ドドスコスコスコドドスコスコスコドドスコスコスコ"は1文字3byteなので、

if len(text) >= 3*24 {} という形にしてやりました。意味としては、24文字 × 一文字分の3byte って感じです。
- text[a:b]
これはズバリ、知識不足の問題っ🌟
スライスですね。これも todo コマンドの時に学びました、が、忘れてました。（というか、詳しく知りませんでした。）
これは、前から、後ろのここまでを持ってくるよー、みたいな感じです。(まだ、全部理解してませんが、そんなイメージです。)
これ、実はPythonにもありました。というか、今回Python版で書いている方のコードをめちゃくちゃ参考にしました（）
参考にしてて思ったことが、GoってめちゃくちゃPythonみたいに書けるなってことです、いや、PythonがGoみたいに書けるのかもしれません...
新しい文法を学ぶ良い機会でした。

最後に。
